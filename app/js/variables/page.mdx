> [!WARNING]
> 
> **Регистр имеет значение!**  
> Переменные с именами `apple` и `APPLE` - две разные переменные.

> [!WARNING]
> 
> **Нелатинские буквы разрешены, но не рекомендуются!**  
> Использование имени `пароль` для переменной не является ошибкой, но крайне не рекомендуется в силу устоявшейся традиции именования переменных на английском языке.  

> [!IMPORTANT] 
>  
> **Зарезервированные имена**  
> Есть [список зарезервированных слов](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Lexical_grammar#keywords), которые нельзя использовать как имена переменных, потому что они используются самим языком.
> Например: `let`, `class`, `break`, `return`, `function` и пр.

## `let`

Используется для объявления изменяемых переменных.

```js
let username; // объявляем переменную       
username = "Tom123"; // присваиваем ей значение
console.log(username); // выводим значение в консоль
// можно объявлять несколько переменных в одной инструкции
let email, birthDate, isActive;
// и сразу при объявлении присваивать значение
let firstName = "Tom", lastName = "Cruise";
```

> [!NOTE]
> 
> **Особенности**
> - `let` имеет блочную область видимости (область действия ограничена фигурными скобками `{}`).
> ```js
> {
> 	let x = 10;
> 	console.log(x); // 10
> }
> console.log(typeof x); // undefined (вне блока x не существует)
> ```
> - Можно объявлять несколько переменных в одной строке.
> ```js
> let a = 1, b = 2, c = 3;
> console.log(a, b, c); // 1 2 3
> ```
> - Переменные, объявленные через `let`, можно изменять.
> ```js
> let count = 5;
> count = 10;
> console.log(count); // 10
> ```

## `const`

Используется для объявления констант - переменных, чье значение не может быть изменено.

```js
const username = "Tom";
// попытка изменить значение вызовет ошибку
username = "Bob"; // ошибка, username - константа, мы не можем изменить ее значение
// константа должна быть инициализирована при объявлении
const email; // ошибка, username не инициализирована
```

> [!NOTE]
> 
> **Особенности**
> - `const` гарантирует, что значение переменной не изменится после её инициализации.
> ```js
> const x = 3;
> x = 5; // Ошибка: Assignment to constant variable
> ```
> - Константы должны быть инициализированы сразу при объявлении, иначе возникнет ошибка.
> ```js
> const age; // Ошибка: Missing initializer in const declaration
> ```

## `var`

Устаревший способ объявления переменных.

```js
var age = 30;
```

> [!TIP]
> 
> **Почему стоит использовать `let` вместо `var`?**
> - `let` является более безопасным и предсказуемым способом работы с переменными.
> - `var` следует использовать только в случае, если вам нужно явно контролировать поведение в функциональной области видимости или если вы понимаете его особенности и намеренно их используете.

> [!NOTE]
> 
> **В чем же разница между `var` и `let`?**
> 1. Область видимости
> - `var` имеет функциональную область видимости (видна внутри всей функции, в которой была объявлена).
> - `let` имеет блочную область видимости (видна только внутри того блока, где была объявлена).
> ```js
> function example() {
> 	if (true) {
> 		var x = 10;
> 		let y = 20;
> 	}
> 	console.log(x); // 10 (var доступен во всей функции)
> 	console.log(y); // Ошибка (y существует только внутри блока)
> }
> example();
> ```
> 2. Поднятие (hoisting)
> - `var` поднимается в область видимости, но не инициализируется, т.е. имеет значение undefined до присваивания.
> - `let` поднимается, но не инициализируется - обращение к переменной до её объявления вызовет ошибку.
> ```js
> console.log(a); // undefined (var поднимается)
> var a = 5;
> 
> console.log(b); // Ошибка (let поднимается, но не инициализируется)
> let b = 10;
> ```
> 3. Дублирование переменной
> - `var` позволяет повторно объявлять переменную в той же области видимости.
> - `let` не позволяет повторно объявлять переменную в той же области.
> ```js
> var c = 1;
> var c = 2; // Нет ошибки
> 
> let d = 3;
> let d = 4; // Ошибка (нельзя переобъявить let)
> ```
